部署 Cloudflare Snippet

2025.10.27 教程

最近上 Cloudflare 的时候发现有一个域名有了 Snippets，这玩意听说用来搭建 Vless 不像 workers 一样会断流。

首先打开你的域名再打开规则这里查看 snippets 这里是不是可以创建

像这个图里面的就可以创建了，如果是显示什么升级 pro 那些就是没有。这玩意好像是看运气给的。

部署 Cloudflare Snippet

第一步：创建并粘贴源码

然后我们复制下面的代码 别忘了再代码第一行里面加上 uuid，uuid 可以在Online UUID Generator Tool生成

然后点击创建片段 名称这里随意 随后粘贴 js 代码

第二步：配置片段触发规则

  1. 完成源码编辑后，点击右上角的  片段规则。
  2. 在规则设置中，选择  自定义筛选表达式，并按以下方式配置：
    • 字段: 选择  主机名。
    • 运算符: 选择  等于。
    • 值: 填写你计划用于此服务的域名（例如  free.frost.cloudns.be，frost.cloudns.be是获得 snippets 资格的域名。理解为在该域名上再一级域名，就像我这个二级域名变成三级域名。请务必替换成你自己的域名，）。
  3. 点击  完成  保存规则

第三步：部署 Snippet 并创建 DNS 记录

  1. 点击页面右上角的  部署  按钮。

  2. 在弹出的确认窗口中，系统会提示  创建新代理 DNS 记录。

  3. 这里域名要和上面的一致，我这里因为教程所以上面截图是在部署好的项目里的

  4. 选择CNAME 类型 ，目标地址选一个优选域名例如www.visa.com。

  5. 最后，点击  创建记录和部署规则。

  6. 稍等片刻，看到右侧显示  活动  状态，部署成功。

第四步：手搓 Vless 节点

现在，我们需要在客户端（如 NekoBox）中手动配置一个 VLESS 节点来连接刚刚部署好的服务。

  • 在 NekoBox 中，参照下图填写必要的连接参数：

    点击服务器，手动输入配置

    • 类型：Vless

    • 名称：为你的节点设置一个自定义名称。

    • 地址 / 端口：地址填写你上面设置的优选域名，端口填写  443。

    • UUID：必须填写你之前在源码中设置的那个 UUID。如果未设置，则可以随意生成

    • 传输协议：选择  ws (WebSocket)。

    • Host：填写你上面设置的子域名。

    • TLS 安全设置这里：勾选不检查服务器证书

    • SNI：填写你上面设置的子域名。

    • ALPN：填 h3

    • 填写完毕后，点击  确定  保存配置

然后就能愉快的使用了

你可能会发现，使用 Cloudflare 部署的节点无法打开某些网站，这通常是因为目标网站也使用了 Cloudflare CDN。

这时候你只需复制节点到人道洛阳花似锦生成优选订阅，就可以通过 proxyip 连接使用有 Cloudflare CDN 的网站了
